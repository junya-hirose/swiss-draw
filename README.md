# Swiss Draw

スイスドロー形式の大会を開くための Next.js プロトタイプです。
admin 側の運営卓と participant 側の参加者画面を持ち、ローカル環境で大会設定、マッチング、結果報告、順位確認まで試せます。

## 起動

```bash
npm run dev
```

ブラウザで開きます。

```text
http://localhost:3000
```

ビルド確認:

```bash
npm run build
```

## 現在の構成

- Framework: Next.js
- UI: React client component
- State: browser `localStorage`
- Main files:
  - `app/page.tsx`
  - `app/globals.css`

現時点ではサーバーDBや認証は未実装です。
同じブラウザ内の別タブ/別ウィンドウでは `localStorage` 経由で大会状態を共有します。

## Admin

### 初期設定

運営側で以下を設定できます。

- 大会名
- 参加者数
  - `~8`
  - `~16`
  - `~32`
  - `~64`
  - `~128`
  - `~256`
  - 自由入力
- 参加者名の入力方式
  - Excelコピー
  - QR参加登録
  - 事前ID参照
- 制限時間
  - `20min`
  - `25min`
  - `30min`
  - `40min`
  - `50min`
  - 自由入力
- BO
  - `BO1`
  - `BO3`
  - `BO5`
  - `BO7`
  - `BO9`
- エリミネーション
  - なし
  - 1敗で脱落
  - 2敗で脱落
  - 3敗で脱落
  - 4敗で脱落
  - 5敗で脱落
- 参加費あり/なし
- 設定の Save / Load
- 参加者URLとQRの発行

参加者名欄に入力された人数に合わせて参加者数は自動補正されます。
例: 20人入力すると `~32` 相当に補正されます。

### 運営中

運営卓では以下を行えます。

- ラウンド開始
- 同ラウンド内の再マッチング
- 全体の Stop / Resume
- テーブルごとの残り時間確認
- テーブルごとの延長
  - デフォルトは `1分`
  - 必要に応じて複数回押す運用
- マッチ結果の入力/修正
- 強制敗北
- ジャッジ対応の記録
  - 注意
  - 警告
  - ゲーム敗北
  - マッチ敗北
  - 失格
- 現ラウンド状況
- 順位表
- ラウンド履歴
- 全マッチ終了後の次ラウンド進行
- 外部連携用データ出力
  - 全結果JSON
  - 順位CSV
  - 対戦CSV
  - JSONコピー

### 戦績表示

各マッチングテーブルには、プレイヤーごとの戦績を `W-L-D` 形式で表示します。

例:

```text
2-0-0
1-1-0
0-2-0
```

この戦績は「ラウンド開始時点」のスナップショットです。
ラウンド中に結果を報告しても、そのラウンドのマッチ表では変わりません。

スイスドロー確認用に、ラウンド開始時点の戦績が異なる組み合わせには `階段` バッジを表示します。

## Participant

参加者側では以下を行えます。

- 自分の参加者ID/名前の確認
- 自分の現在マッチ確認
- 自分のテーブルの残り時間確認
- BO設定に合わせた結果報告
- Reload
- 現在順位の確認
- 過去ラウンド結果の確認
- 大会結果の確認

参加者URLは admin 側で発行されます。

```text
http://localhost:3000?view=participant&event=EVENT_CODE&player=PLAYER_ID
```

## スイスドローの現状仕様

- 同じ勝敗帯を優先してペアリングします。
- 過去に対戦した相手との再戦を避けます。
- 奇数人数の場合は BYE を付与します。
- エリミネーション設定に応じて、規定敗北数以上のプレイヤーは以降のマッチング対象から外れます。
- 失格プレイヤーも以降のマッチング対象から外れます。
- 推奨スイスラウンド数を参加者数から初期設定し、運営側で手動修正できます。
- 順位はマッチポイントと暫定 OMW% を表示します。
- Admin/Judge 操作として両者敗北を記録できます。
- 大会結果は JSON / 順位CSV / 対戦CSV として保存できます。
- JSON には大会設定、参加者、チェックイン、デッキ登録、順位、全ラウンド結果、裁定履歴を含みます。

## 現在の制限

このプロジェクトはまだローカルプロトタイプです。

- 永続DBなし
- ログイン/権限管理なし
- participant URL の不正アクセス対策なし
- 複数端末間のリアルタイム同期なし
- 本物のQRコード生成ライブラリ未導入
- 引き分け入力は未実装
- 第3・第4タイブレーカーなど、詳細な最終順位計算は未実装

本番運用に近づける場合は、DB/API、認証、WebSocket/SSE、監査ログ、正規のQR生成を追加する想定です。
